Тип файлов принимаемых формой, должен быть в .txt формате. 
Т.к. приложение тестовое, оставлен только этот тип, для примера работы. 

В случае если будет выбран файл другого типа, появится всплывающее окно с предупреждением. 

Если выбран текстовый файл с данными, его можно сохранить на сервер. 

Вся логика работы приложения в двух файлах:
add_file.js - клиентская часть.
json.php - серверная часть.
Взаимодействие между ними осуществляется с помощью объекта "XMLHttpRequest". 
Данные от серверной части, в js-файл, передаются в виде json-строки. 

После компрессинга и декомпрессинга полученного файла, он добавляется в zip-архив.  
И далее копируется в папки на сервер. 
В виде упакованного и не упакованного вариантов, в папку "upload_file". 
Также в виде архива с двумя файлами, в папку "upload_zip". 

Под формой, после записи файла на сервер, появятся плавно 3 кнопки. 
С помощью них можно скачать, два варианта(упакованный, не упакованный) загруженного файла. И архив с данными файлами. 
Также появится кнопка на обновление страницы. 

Дополнительно, на кнопке для загрузки, выводится несколько разных состояний загрузки файла. 
И итоговое кол-во загруженных кб, на сервер.

В архив с приложением, добавлен дополнительно файл .htaccess, с директивой "AddDefaultCharset utf-8".
В случае если при загрузке в IE, страницы с сервисом будет ошибка кодировки. 

Еще файлы приложения:
index.html - разметка страницы на bootstrap3. Форма для выбора файла.
css/style.css - стили приложения.
